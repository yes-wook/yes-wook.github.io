## 안녕하세요, 김정욱입니다

데이터가 팀의 의사결정과 제품의 운영을 안정적으로 뒷받침하도록 만드는 **Data Platform Engineer**입니다.

데이터 파이프라인을 만들 때는 적재 성공만 보지 않습니다. 데이터가 어디에서 왔는지 설명할 수 있는지, 실패했을 때 다시 실행할 수 있는지, 권한과 비용을 통제할 수 있는지, workload가 달라졌을 때 성능 판단을 다시 할 수 있는지를 함께 설계합니다.

## 관심 분야

- Delta Lake와 Lakehouse 운영
- Spark 기반 대용량 데이터 처리와 성능 최적화
- 증분 파이프라인, 데이터 품질, 관측성
- 분석 플랫폼의 보안·비용·셀프서비스 설계
- 기술 선택을 측정 가능한 운영 기준으로 바꾸는 일

## 이 사이트에서 다루는 것

이곳에는 실무에서 얻은 기술 판단과 운영 원칙을 익명화해 기록합니다. 특정 기능을 무조건 추천하기보다, 어떤 workload에서 어떤 비용과 제약을 감수했는지, 이후 환경이 바뀌었을 때 무엇을 다시 검증했는지를 중심으로 씁니다.
