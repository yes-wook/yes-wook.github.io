<p><em>Databricks Runtime 변경이 가져온 에러</em></p><h3>1. 문제 상황: 평소 잘 되던 배치가 갑자기 실패하다</h3><p>최근 TV 적재 테이블의 일배치 작업에서 아래와 같은 에러가 발생했습니다.</p><pre>The value &#39;&#39; of the type &quot;STRING&quot; cannot be cast to &quot;TIMESTAMP&quot; because it is malformed.<br>Correct the value as per the syntax, or change its target type.<br>Use `try_cast` to tolerate malformed input and return NULL instead.</pre><p>이 테이블에서는 기존에 특별한 이슈가 없었는데, 갑자기 오류가 발생했습니다.<br>원인을 파악하기 위해 적재 파이프라인을 하나씩 뜯어봤습니다.</p><h3>2. 원인 분석: Spark 4.0의 동작 변화</h3><p>최근 Databricks Runtime을 교체하면서 Spark 버전이 <strong>3.x → 4.x</strong>로 올라갔는데,<br>이 과정에서 <strong>CAST 동작이 변경</strong>된 것이 원인이었습니다.</p><p>즉, 기존에는 잘 넘어가던 잘못된 값(&#39;&#39;)이 Spark 4.x에서는 <strong>강제 에러</strong>로 처리된 것입니다.</p><ul><li><strong>적재 방식</strong>: Parquet → Delta Table 변환</li><li><strong>문제 칼럼</strong>: STRING → TIMESTAMP로 CAST</li><li><strong>Spark 3.x 동작</strong>: CAST 실패 시 자동으로 NULL 처리</li><li><strong>Spark 4.x 동작</strong>: CAST 실패 시 <strong>에러 발생</strong></li></ul><h3>3. Spark 4.0의 변화: 데이터 무결성과 개발자 생산성 강화</h3><p>Spark 4.0은 단순한 성능 개선을 넘어, <strong>데이터 무결성</strong>과 <strong>개발자 생산성</strong>을 크게 향상시키는 업데이트를 도입했습니다.</p><p>특히 중요한 변화로 <strong>ANSI SQL 모드 </strong>기본 활성화가 있습니다.</p><h4>ANSI SQL 모드란?</h4><p>쉽게 말해, SQL 표준을 따르는 것입니다.</p><ul><li><strong>이전 동작</strong>: 숫자 오버플로우, 0으로 나누기, 잘못된 CAST → 조용히 NULL 반환</li><li><strong>Spark 4.0 동작</strong>: 명확한 오류 메시지 제공 → 데이터 품질 보장</li></ul><p>이 변화는 다음과 같은 장점을 제공합니다:</p><ul><li><strong>데이터 무결성 강화</strong>: 잘못된 값이 조용히 넘어가지 않고, 명확한 오류로 드러남</li></ul><p>결과적으로, Spark 4.0은 <strong>더 명확하고, 신뢰할 수 있으며, 이식 가능한 데이터 워크플로우</strong>를 촉진합니다.</p><h3>4. 해결 방법: TRY_CAST로 안전하게</h3><p>에러 해결은 간단했습니다. 아래처럼 함수를 변경하였습니다.</p><ul><li>CAST → <strong>TRY_CAST</strong>로 변경<br> → 실패 시 NULL 반환</li></ul><p>운영 환경에서 변경하실 때는, null비율을 모니터링하며 너무 과하게 활용되는건 아닌지 확인하는게 좋습니다.</p><h3>5. 마무리</h3><p>Spark 4.0으로 올라가면서 SQL 함수의 동작이 조금씩 바뀌고 있습니다.<br> 이번 사례는 그 변화가 실무에 어떤 영향을 줄 수 있는지 보여줍니다.</p><p>💡 <strong>핵심 메시지</strong>:<br> “버전 업그레이드 시, CAST 동작처럼 사소해 보이는 변경이 대규모 데이터 적재에서 장애를 유발할 수 있다.</p><ul><li><a href="https://docs.databricks.com/aws/en/sql/language-manual/parameters/ansi_mode">ANSI_MODE | Databricks on AWS</a></li><li><a href="https://www.databricks.com/kr/blog/introducing-apache-spark-40">Apache Spark 4.0 소개</a></li></ul><p><a href="https://docs.databricks.com/aws/en/release-notes/runtime/">https://docs.databricks.com/aws/en/release-notes/runtime/</a></p>
